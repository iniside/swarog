# Historical docs are archives, not current-state specifications

Plans, dated reviews, status reports, and summaries in `docs/` preserve what was
known and decided at that time. Do not flag their superseded statements as current
documentation drift and do not propose rewriting them to match HEAD. Use them to
reconstruct why the architecture changed. Derive current state from the code,
`CLAUDE.md`, and explicitly current reference documentation. Only report a dated
document when it falsely presents itself as current operational guidance.
