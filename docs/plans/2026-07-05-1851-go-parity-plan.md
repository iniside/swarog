# Plan: parity backendu Go z Quarkus-sketchem (dual-deploy + QUIC edge)

> **Nowy plik w repo (na zatwierdzeniu):** `docs/plans/2026-07-05-1851-go-parity-plan.md`.
> **Status:** przeszedł grumpy-review (Opus, ultrathink). Blocker-y B1–B3 + S1–S7 wchłonięte; log rozwiązań na końcu.
> Duży, wielokrokowy refactor; każdy krok buduje i weryfikuje. Marquee (round-trip po QUIC) wcześnie (Krok 2).

## Context

Sesja zważyła Go vs Kotlin. Na osi „jedna binarka, ROLES wybiera monolit-albo-mikroserwisy + transport-transparent
seamy + QUIC" **Go jest lżejsze**: QUIC natywnie (`quic-go`, bez FFM/msquic), przełącznik topologii jawny w
`main()` (bez CDI-magii = pół sesji runtime-bugów), rdzeń **nietknięty** bo szwy już istnieją. Plan reprodukuje w
backendzie Go (root) pełną funkcjonalność `experiments/jvm-quarkus-sketch/`.

**Teza:** parity **additywne** — nowe pakiety + wiring w `cmd/server/main.go` + zmiany w modułach/kontraktach,
**bez edycji `core/`** (CLAUDE.md #1). Role-parsing ląduje w `cmd`, nie `core` (żeby teza była dosłowna).

## Stan zastany (zweryfikowany z kodu)

- **`core/module.go`:** `Module{Name,DependsOn,Init}`, opcj. `Migrator/Starter/Stopper`; `Registry` (topoSort:
  cycle/missing-dep, `Add` **panikuje na duplikat Name**); `Context{Bus,Mux,DB,Log,services,contributions}`;
  `Provide(name,any)` **panikuje na duplikat**/`Require(name)any` (konsument asertuje do WŁASNEGO interfejsu).
  `Contribute`/`Contributions`.
- **`core/bus.go`:** `Define[T]/Emit/On`, **async fire-and-forget**, per-sub goroutine+mailbox, drain na `Close`.
- **Sync seam:** `characters.go:140` `service.OwnerOf(ctx, id string)(playerID string, ok bool)`, provided
  `"characters"`. Konsument `inventory.go:27-29` `charactersSvc{OwnerOf}` + `Require("characters")`. Też
  `accounts` service `VerifySession(ctx,token)(playerID string,ok bool)` (auth), Require'owany przez characters+inventory.
- **Async seam:** characters `core.Emit(...CreatedEvent/DeletedEvent)` (`characters.go:95,133`) — **PO** `store.create`,
  NIE w tx. inventory `core.On(...CreatedEvent, onCharacterCreated)` → grant startera / wipe (`inventory.go:76-77,93-104`).
- **Admin:** `adminapi.Item{Section,Label, Render func(ctx)(Content,error)}` — **closure**. 3 moduły kontrybuują;
  admin czyta `Contributions` i woła `Render` in-process.
- **main.go:** bezwarunkowy `reg.Add` ×8, port `:8080` hardcoded, brak `/healthz`.
- **Persistence:** raw pgx, per-module `Migrate` (raw DDL), schema-per-module. Schema-split **już izolowany**.
- **accounts stan:** `epic_oauth.go:34` in-memory `states map` (OAuth authorize→session) — **NIE stateless**.
- **Brak:** ról, remote, outbox, admin fan-out, health, quic-go/config-lib/prometheus.

## Topologia splitu (zrewidowana — dodge B3, dwa remote seamy przez jeden edge)

- **Proces A** `ROLES=accounts,characters` HTTP :8080, edge/QUIC :9000 — hostuje accounts+characters (real);
  **serwer edge** wystawia `accounts.verifySession` **i** `characters.ownerOf`; **producent** outbox (character.*).
  Epic-OAuth (`/accounts/epic/*`) żyje TYLKO tu (in-memory state map bezpieczny — jeden proces).
- **Proces B** `ROLES=inventory,admin` HTTP :8081 — hostuje inventory+admin (real); **stuby** `accounts`+`characters`
  Provide'ują remote-klienty edge → A (jedyne remote sync seamy); **sink** `/events/*` (synchronous handler);
  admin fan-out (inventory local-closure + characters remote HTTP `/admin-data`).
- **Monolit** `ROLES` unset → wszystko lokalnie, :8080, bez QUIC/certu/stubów (gałąź local, bus in-process).

**Biblioteki (2026):** `quic-go` v0.60 (pin+verify API), codec stdlib `encoding/json` (za `Codec` iface; msgpack
później), cert in-memory `crypto/ecdsa`+`x509` (bez OS store). Opcjonalne/odroczone: `caarlos0/env`, prometheus,
sqlc, goose — NIE blokują parity.

### Krok 1 — Role + gating + port env + /healthz `[opus]`
- **(a)** `cmd/server/main.go`: `parseRoles(os.Getenv("ROLES"))`→`roleSet` (helper w `cmd`, NIE core — N1); unset=all.
  **Explicit stub-computation (S3):** `hosted = roleSet` (nazwa roli == nazwa modułu); `needed = ⋃DependsOn(hosted)
  \ hosted`; dla każdego `m∈hosted` → `reg.Add(realModule[m])`; dla każdego `n∈needed` → `reg.Add(remoteStub[n])`
  (dokładnie JEDEN; assert real∧stub rozłączne — inaczej `Add` panikuje). Port z env `PORT` (default `:8080`);
  `mux.HandleFunc("GET /healthz", ...)` (200 + `db.Ping`).
- **(a) remote-stub (Krok 1 = szkielet, klient w Krok 3):** `<dep>remote.Module{peerAddr}` impl `core.Module`:
  `Name()="<dep>"`, `DependsOn()=nil`, `Init`=`ctx.Provide("<dep>", client)`. Bez Migrate/routes; jest `Stopper`
  (zamyka conn). topoSort: stub(nil-deps) i realne fundamenty przed dependentami — Provide-before-Require
  gwarantowane (dependent Init biegnie po). W Kroku 1 klient = placeholder; Krok 3 podstawia edge-klient.
- **(b)** Fundament; port+health potrzebne skryptowi (Krok 6).
- **(c)** ZERO zmian `core/`. Schema-split za darmo. Uwaga N2: w B `/characters/*`,`/accounts/*` = 404 (routes tylko
  na A) — udokumentować w topologii.
- **(d)** `[opus]`.
- **Weryfikacja:** `go build/vet`; monolit jak dziś; `ROLES=accounts,characters` = podzbiór, migruje tylko te
  schematy, `/healthz`=200 na env-porcie. (Krok 1 testuje gating czysto-lokalnymi podzbiorami; remote w Krok 3.)

### Krok 2 — QUIC RPC core (`edge/`) + cert `[opus]`  ← MARQUEE
- **(a)** Pakiet `edge/` (top-level, transport-agnostyczny): `Codec` iface (`Encode/Decode`)+`jsonCodec`; framing
  4B-BE-len+payload na `quic.Stream`; `Server{ListenAddr(tlsConf,quicConf); Handle(method, fn)}` — `Accept` conn
  →`AcceptStream`→ ramka req→dispatch po `method`→ramka resp; `Client` trzyma **persistent `*quic.Conn`**
  (`DialAddr` raz), `Call(method,req,resp)`=`OpenStreamSync` świeży stream per call, write/read, close stream (S5).
  `Client` jest zamykalny (`Close` → conn.CloseWithError). Cert `edge/tls.go` in-memory self-signed
  (`ecdsa.P256`+`x509`, `tls.Config.NextProtos=["edge"]` **po OBU stronach** — N4; klient `InsecureSkipVerify`).
- **(b)** Transport PRZED seamem — wczesny dowód.
- **(c)** `quic-go` do go.mod (pin v0.60, zweryfikuj `Conn`/`Listener.Accept`/`DialAddr(ctx,addr,tls,quic)` — S5).
  Persistent conn + stream-per-call (nie re-dial per call!). Codec za iface. Bez OS store.
- **(d)** `[opus]`.
- **Weryfikacja:** `edge/edge_test.go` — serwer+klient po **realnym QUIC/UDP** localhost, round-trip typed call +
  błąd + (opcj.) równoległe calle na jednym conn. `go test ./edge/`. Odpowiednik Kotlinowego echo-testu.

### Krok 3 — Transport-transparent sync seams po QUIC (`ownerOf` + `verifySession`) `[opus]`
- **(a) Poszerz interfejsy o error (B2):** `characters.service.OwnerOf(ctx,id)(string,bool,error)` i
  `accounts...VerifySession(ctx,token)(string,bool,error)`; zaktualizuj konsumenckie interfejsy (`inventory
  .charactersSvc`/`accountsSvc`, `characters.accountsSvc`) i call-sites (`inventory.go:122` — network-err→**503**,
  genuine `!ok`→404/403; local zwraca `nil` error).
- **(a) Serwer (A):** gdy rola hostuje accounts/characters — Starter startuje `edge.Server` na `EDGE_ADDR`
  (:9000), Handle `"characters.ownerOf"` (`{ID}`→`{PlayerID,Ok}`) i `"accounts.verifySession"`
  (`{Token}`→`{PlayerID,Ok}`) delegujące do local service. Gated: aktywne ∧ nie-monolit.
- **(a) Stuby (B):** `charactersremote`/`accountsremote` Provide'ują klienty: `OwnerOf`/`VerifySession` =
  `edge.Client.Call(...)` z reconnect-on-conn-failure (jeden retry), a twarda awaria → **zwróć error** (B2), nie
  `("",false)`. Stub jest `Stopper` (zamyka `edge.Client`). main() rejestruje stub zamiast real gdy rola nie hostuje.
- **(b)** Drugi seam; zależy od 1 (stub/gating) i 2 (edge).
- **(c)** `Require(...).(iface)` w konsumentach — sygnatury zmienione o `error`, ale local i remote identyczne.
  Cert thumbprint NIE potrzebny (Go in-memory cert). Persistent conn (Krok 2).
- **(d)** `[opus]`.
- **Weryfikacja:** split (A :8080 edge:9000 / B :8081, EDGE_ADDR=localhost:9000). Utwórz postać na A (auth na A);
  `GET /inventory/character/{id}` na B → `characters.OwnerOf` + `accounts.VerifySession` **po QUIC** do A: realna=200,
  cudza=403, nieistniejąca=404; **zabij A → 503** (nie 404). Monolit: local, bez QUIC.

### Krok 4 — Async: outbox + relay + synchronous sink + inbox `[opus]`
- **(a) Outbox (characters):** DDL `characters.outbox(id BIGSERIAL pk, topic, payload jsonb, created_at, sent_at
  null)`. Restrukturyzuj create/delete: `store.create`+outbox-insert w JEDNEJ tx (`BeginTx`), po commit `core.Emit`
  lokalnie (współlokowani subskrybenci). **S4 udokumentuj:** crash po commit przed Emit gubi LOKALNĄ dostawę
  (outbox ratuje tylko remote) — akceptowalne, zgodne z dotychczasową best-effort semantyką busa.
- **(a) Relay (Starter, gated rolą-właścicielem):** ticker ~500ms; `SELECT ... WHERE sent_at IS NULL ORDER BY id
  FOR UPDATE SKIP LOCKED`; **per-subscriber ordering (S6):** POST-uj rosnąco po id, **zatrzymaj się na pierwszym
  błędzie do danego subskrybenta** (nie przeskakuj — inaczej delete-before-create); w POST wyślij **stabilny
  event_id = outbox.id** (S6) w body/nagłówku. Mark `sent_at` po 2xx. Zero subów dla topiku → mark sent od razu.
  Helper w pakiecie `outbox/` (nie core). Config subskrybentów: `EVENTS_SUBSCRIBERS` per-topic (jak Kotlin).
- **(a) Sink + inbox (inventory) — SYNCHRONICZNY (B1):** `POST /events/character-created`/`-deleted`: dekoduj
  JSON + event_id; w JEDNEJ tx `INSERT inventory.inbox(event_id) ON CONFLICT DO NOTHING` (0 wierszy→już→200) →
  **wywołaj logikę grant/wipe SYNCHRONICZNIE** (ta sama funkcja co `onCharacterCreated`, w module inventory — brak
  #2) → commit → **dopiero teraz 200**. NIE re-emituj na bus (async ack byłby kłamstwem — B1). DDL
  `inventory.inbox(event_id text pk, processed_at)`.
- **(b)** Trzeci seam; zależy od 1.
- **(c)** Bus ZOSTAJE in-process, ale **tylko monolit/co-located** path (`core.On`). Split: emit lokalny w A
  (brak subskrybenta) + outbox→relay→**sink synchroniczny w B**→grant. Ta sama funkcja grant wołana przez bus
  (monolit) ALBO sink (split) — **dokładnie jedna per topologia** (subskrybent w jednym procesie, brak
  double-delivery). Idempotencja: inbox dedup + efekt atomowo, ack po sukcesie.
- **(d)** `[opus]` — tx/idempotencja/ordering/ack.
- **Weryfikacja:** split — postać w A → outbox → POST do B sink → grant (synchronicznie, 200 po grancie); podwójny
  POST → jeden grant (inbox); zabij B mid-POST → outbox retry (sent_at null). Monolit — natychmiastowy grant
  (local bus), outbox drenuje do nikogo.

### Krok 5 — Admin fan-out (local-closure + remote-URL, NIE usuwanie Render) `[opus]`
- **(a)** `adminapi.Item`: dodaj `ID string`; zachowaj `Render func(ctx)(Content,error)` (**local, in-process** —
  S1/S2) ORAZ dodaj wariant remote: `RemoteURL string` (ustawiany tylko przez stub). Real moduł kontrybuuje
  `Item{ID,Section,Label,Render:closure}` + wystawia `GET /admin-data/<id>` (JSON `Content`, ta sama logika co
  `adminSection`). Stub (dla remote modułu widocznego w adminie) kontrybuuje `Item{ID,Section,Label,RemoteURL:
  peer/admin-data/<id>}` (bez Render). `modules/admin/admin.go`: dla każdego `Item` — `Render!=nil` → wołaj local;
  wpp `RemoteURL` → HTTP GET → dekoduj `Content`; per-item try/err → error-card. Sortuj (Section,Label), slug jw.
- **(b)** Admin to jedyny seam z closure-assumption; parity wymaga fan-outu. Rozwiązuje S1 (local=closure, brak
  cross-module import) i S2 (stub kontrybuuje metadata-item z RemoteURL → źródło sidebaru jednolite: `Contributions`).
- **(c)** ownerOf/verifySession=QUIC, ale admin-data=REST/JSON świadomie (bogaty display, admin i tak HTTP).
  `Content/KPI/Table/Cell` już serializowalne. Discovery przez stub (peer URL z configu main()) — bez osobnego
  `ADMIN_MODULES`.
- **(d)** `[opus]` — redesign kontraktu.
- **Weryfikacja:** split — `/admin` w B: Inventory (local closure) + Characters (remote HTTP `/admin-data/characters`
  do A, przez stub-item); zabij A → characters=error-card, inventory żyje. Monolit: wszystko local, identyczny wygląd.

## Ryzyka / świadome
- **Dwa remote seamy** (ownerOf+verifySession) przez jeden edge — B płaci remote-auth-hop per request (koszt
  splitu auth; akceptowalny dla reference).
- **Emit-tx restrukturyzacja** (create+outbox w tx) — uważać na ścieżki błędu; local delivery best-effort (S4).
- **quic-go v0.60 API** — pin + zweryfikuj przy implementacji.
- **Rozmiar** — 7 kroków; marquee (QUIC round-trip) w Kroku 2.
- **`core/` nietknięty** (poza niczym — parseRoles w cmd) — jeśli krok kusi edycją core, to sygnał błędu.
