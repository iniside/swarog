## Inwarianty (każdy krok)

Zero-sharing; single stop-authority (`STOP` sygnał ⊥ `fleet_stop` control); pure
policy fns (`step`/`stop_outcome`/`status_of`) zostają czyste (no I/O, clock-injected);
**weles never builds**; readiness ⊥ restart. **NOWY (A1):** żadna zmiana nie może
respawnować usługi, której zabicia NIE potwierdzono (unconfirmed kill ⇒ Failed, nie
Backoff) — inaczej dublujemy proces nad sierotą.

### Step 1 — A1: Kill-path domyka autorytet shutdown-Err `[opus]` core-implementer
**(a) Co:** `weles/src/supervisor.rs`.
- **Nowa CZYSTA fn** obok `stop_outcome` (`:276`) / `status_of` (`:253`):
  ```rust
  /// A hung service's kill was requested. If the stop was CONFIRMED
  /// (`stop_outcome`'s clean bit), adopt the intended post-kill phase; if it was
  /// UNCONFIRMED (Err — force could not confirm the exit, the process may be
  /// ORPHANED and still holding its port), give up on the service (`Failed`)
  /// instead of `Backoff`, so the monitor NEVER respawns a second instance over
  /// a possible orphan. Reuses the ONE authority (`stop_outcome`) for "confirmed
  /// gone"; layers the Kill-loop policy on top.
  fn phase_after_kill(shutdown: &Result<Outcome>, intended: Phase) -> Phase {
      match stop_outcome(shutdown) {
          (_, true) => intended,
          (_, false) => Phase::Failed,
      }
  }
  ```
- `Directive::Kill` arm (`:1042-1054`) — zamiast połykać `Err` i bezwarunkowo
  przyjmować `new_phase`, przechwycić wynik i przepuścić przez `phase_after_kill`:
  ```rust
  Directive::Kill(new_phase) => {
      eprintln!("weles: {name} did not become healthy within {HEALTH_DEADLINE:?} — stopping it");
      let new_phase = match svc.proc.take() {
          Some(mut proc) => {
              let result = proc.shutdown(HUNG_GRACE, HUNG_FORCE);
              if let Err(error) = &result {
                  eprintln!(
                      "weles: stopping hung {name} failed: {error:#} — the process may be \
                       ORPHANED (force could not confirm it exited); giving up on {name} \
                       (NOT respawning over a possible orphan) — check for a stray {name}"
                  );
              }
              phase_after_kill(&result, new_phase)
          }
          None => new_phase, // observe() found it Alive, so this is unreachable; stay defensive
      };
      svc.phase = Some(new_phase);
      svc.status = status_of(new_phase);
      true
  }
  ```
  (Warning-tekst ujednolicony z teardownem `:1091-1095` — „ORPHANED", ale z dodatkiem
  „giving up / NOT respawning" bo tu polityka jest inna niż terminal-stop.)

(dublowanie procesu), a Step 2 (P1 test) dotyka tego samego pliku i statyka `STOP`.
